## API Architecture
- **Single internal API**: All data flows through `api.m.jd.com/?functionId=qryCompositeMaterials`
- **h5st signing**: Every API call requires an `h5st` signature generated by `window.PSign.sign()` in the browser context
- `PSign` uses `appId: "e56d4"` and generates time-bound HMAC signatures
- Different content modules are fetched by passing different `type` (productGroup/advertGroup) and `id` in the `qryParam` body
- Module IDs from `window.serverIdList`: recommend=12240015, banner=05382124, squareLeft=06813315, squareRight=06813381, messages=06784803
- Only `productGroup` type returns actual product data; all `advertGroup` types return banner/ad content

## Auth
- No auth needed for public data on global.jd.com
- `requires_auth: false`
- Note: The main JD site (search.jd.com, item.jd.com) requires login for all functionality

## Known Limitations
- **No search API**: The `search` functionId returns "no access" regardless of signing. `global.jd.com/search` routes to `search.jd.com` which requires login.
- **No product detail API**: `item.jd.com` requires login. `global.jd.com/product/{skuId}.html` renders the homepage shell without product data.
- **No review API**: Product reviews are only available on the main JD site (requires login).
- **Bot detection**: `search.jd.com` has aggressive bot detection (redirects to `cfe.m.jd.com/privatedomain/risk_handler/`). The main JD domain also redirects international traffic to `corporate.jd.com`.
- **Session fragility**: Login sessions on `global.jd.com` are lost when navigating to other JD subdomains (`passport.jd.com`, `item.jd.com`).
- **Recommendation-only products**: Product data comes exclusively from the recommendation module (productGroup 12240015). No category-specific or keyword-filtered product queries available.
- **Fixed product set**: Recommendations return ~37 products. Pagination parameter has no effect (same set returned regardless of page number).
