# JD.com

## Overview
E-commerce. JD.com (京东) is China's #2 e-commerce platform. Covers two tiers:
- **Public** (global.jd.com): Product recommendations, promotions, categories — no login needed
- **Logged-in** (search/item/cart.jd.com): Product search, detail, reviews, cart — requires login

Uses h5st-signed internal API via `api.m.jd.com` and DOM extraction.

## Operations
| Operation | Intent | Method | Notes |
|-----------|--------|--------|-------|
| getRecommendations | homepage product recommendations | GET /internal/getRecommendations | ~37 products with skuId, name, 3 price tiers, shop info, ratings |
| getPromoBanners | promotional banner ads | GET /internal/getPromoBanners | hero banner images with campaign links |
| getSquarePromotions | square promotional modules | GET /internal/getSquarePromotions | campaign banners in left/right layout |
| getNewsMessages | site news/announcements | GET /internal/getNewsMessages | shipping updates, policy notices |
| getCategoryNavigation | category navigation tree | GET /internal/getCategoryNavigation | DOM extraction of category sidebar |
| searchProducts | search products by keyword | GET /internal/searchProducts | 30 results/page, paginated, requires login |
| getProductDetail | product detail by SKU ID | GET /internal/getProductDetail | name, price, images, variants, review summary, requires login |
| getProductReviews | product reviews by SKU ID | GET /internal/getProductReviews | review count, good rate, individual review text, requires login |
| getCart | shopping cart contents | GET /internal/getCart | item names, prices, shops — read only, requires login |
| getHotSearchWords | trending search keywords | GET /internal/getHotSearchWords | popular search terms on JD, requires login |

## API Architecture

### Global (public)
- **Single internal API**: All data flows through `api.m.jd.com/?functionId=qryCompositeMaterials`
- **h5st signing**: Every API call requires an `h5st` signature generated by `window.PSign.sign()` in the browser context
- `PSign` uses `appId: "e56d4"` and generates time-bound HMAC signatures
- Different content modules fetched by `type` (productGroup/advertGroup) and `id` in `qryParam` body
- Module IDs from `window.serverIdList`: recommend=12240015, banner=05382124, squareLeft=06813315, squareRight=06813381, messages=06784803

### Main site (logged-in)
- **Search API**: `api.m.jd.com/api?functionId=pc_search_searchWare` with `appid=search-pc-java` — works with cookies, no h5st needed
- **Product detail**: Extracted from `window.pageConfig.product` on item.jd.com/{skuId}.html
- **Reviews**: DOM extraction from item.jd.com product pages
- **Cart**: DOM extraction from cart.jd.com
- **Hot words**: `api.m.jd.com/api?functionId=pc_search_hotwords` with `appid=search-pc-java`

## Auth
- **Public ops** (recommendations, promos, categories, news): No auth needed — `global.jd.com`
- **Logged-in ops** (search, detail, reviews, cart, hot words): Requires JD login cookies
- `requires_auth: optional` — public ops work without auth, search/detail/reviews/cart need login

## Transport
- `transport: page` — L3 adapter for all operations
- h5st signing performed in browser context via `window.PSign.sign()`
- Search API uses cookies only (no h5st) from search.jd.com context
- Product detail/reviews use DOM extraction from item.jd.com
- Cart uses DOM extraction from cart.jd.com
- Direct HTTP to `api.m.jd.com` without browser context returns "no access"

## Extraction
- L3 adapter: `jd-global-api`
- Global ops: `page.evaluate()` → `PSign.sign()` → `fetch(api.m.jd.com)` → JSON
- Search: `page.evaluate()` → `fetch(api.m.jd.com/api?functionId=pc_search_searchWare)` → JSON
- Product detail: `page.evaluate()` → `window.pageConfig.product` + DOM price/review data
- Reviews: `page.evaluate()` → DOM text extraction from comment section
- Cart: `page.evaluate()` → DOM text extraction from cart items
- Category navigation: DOM extraction from homepage sidebar

## Known Limitations
- **www.jd.com redirects**: International traffic on `www.jd.com` redirects to `corporate.jd.com`. Use `global.jd.com` for public data.
- **Session fragility**: Login sessions may break when navigating between JD subdomains (`global.jd.com` ↔ `search.jd.com` ↔ `item.jd.com`).
- **Bot detection**: `search.jd.com` has aggressive bot detection. Works with logged-in session but may trigger CAPTCHA under heavy use.
- **Review extraction**: Reviews are extracted from DOM text — limited to the first few reviews shown on the product page (typically 1-5).
- **Cart extraction**: Cart items are parsed from DOM text. Complex pricing (coupons, bundles) may not be fully captured.
- **Fixed recommendations**: Recommendation module returns ~37 products. Pagination has no effect.
- **m.jd.com search**: Mobile JD search also requires login.

## Pricing Model
Three price tiers per product (recommendations):
- `jdPrice` — current JD listed price
- `originalPrice` — original/suggested retail price
- `promotionalPrice` — promotional/discounted price (lowest)

Search results show `jdPrice` and `originalPrice`.
Product detail shows the current price from DOM.
