## Workflows

### Search for flights
1. `searchFlights(origin, destination, departureDate, returnDate)` â†’ results with `resultId`, `legs`, `segments`
2. Results include booking URLs to provider sites (Southwest, United, etc.)
3. Use `legs` map to get duration/times, `segments` map for airline/flight details

### Search for hotels
1. `searchHotels(destination, checkInDate, checkOutDate)` â†’ results with hotel name, rating, prices
2. Results include prices from multiple providers for comparison

### Search for rental cars
1. `searchCars(location, pickupDate, dropoffDate)` â†’ results with vehicle type, agency, price
2. Use `agencyMap` and `providerMap` to resolve provider details

## Operations

| Operation | Intent | Key Input | Key Output | Notes |
|-----------|--------|-----------|------------|-------|
| searchFlights | find flights | origin, destination, departureDate | results, legs, segments, airlines | round-trip if returnDate given |
| searchHotels | find hotels | destination, checkInDate, checkOutDate | results with provider pricing | use dashes in destination |
| searchCars | find rental cars | location, pickupDate, dropoffDate | results, agencyMap, providerMap | airport code or city |

## Site Internals

## API Architecture
- Internal REST APIs behind progressive polling pattern
- Flights: `POST /i/api/search/dynamic/flights/poll`
- Hotels: `POST /i/api/search/dynamic/hotels/poll`
- Cars: `POST /i/api/search/v1/cars/poll`
- All require JS-generated `x-csrf` header and server-assigned `searchId`

## Auth
- No authentication required for search (public)
- CSRF token generated by client-side JS (not from cookies or meta tags)
- Session cookies: `kayak`, `kmkid`, `csid`, `p1.med.sid`

## Transport
- **page** transport required (adapter/intercept pattern)
- Akamai bot detection present â€” direct Node HTTP will be blocked
- Adapter: `adapters/kayak-search.ts`
- Pattern: navigate to search URL â†’ intercept poll API responses
- Poll is progressive: results accumulate across multiple responses

## Bot Detection
- Akamai Bot Manager (`_abck` cookie present on some subdomains)
- Cloudflare (`cf_clearance` on related domains)
- Forter fraud detection (`forterToken` cookie)
- Page transport handles this via real Chrome execution

## Known Limitations
- Search results depend on Kayak's progressive polling â€” adapter waits up to 30s
- Hotel URLs need specific format: city name with dashes or city,state-cID
- Car search location resolution may redirect (e.g., city name â†’ nearest airport)
- Booking URLs are redirects to external provider sites
