# Xiaohongshu (小红书)

## Overview
Chinese social media/lifestyle platform (Instagram + Pinterest hybrid). Archetype: Social Media.

## Operations
| Operation | Intent | Method | Notes |
|-----------|--------|--------|-------|
| searchNotes | search posts by keyword | Navigate to /search_result?keyword= | Extracts from Vue SSR state `__INITIAL_STATE__.search.feeds` |
| getNoteDetail | get post detail + comments | Navigate to /explore/{noteId} | Note from SSR state, comments intercepted from async API |
| getUserProfile | get user profile | Navigate to /user/profile/{userId} | Extracts from `__INITIAL_STATE__.user.userPageData`. May trigger CAPTCHA under rate limiting |

## API Architecture
- Vue 3 SSR app — data baked into `window.__INITIAL_STATE__` on page load
- Internal REST APIs exist at `edith.xiaohongshu.com/api/sns/web/v1/*` and `/api/sns/web/v2/*`
- **Direct API calls fail (500)**: All endpoints require anti-bot signature headers (`X-s`, `X-t`, `X-s-common`) generated by obfuscated JavaScript from `as.xiaohongshu.com/api/sec/v1/scripting`
- Comments loaded async via `edith.xiaohongshu.com/api/sns/web/v2/comment/page` — intercepted from page's own JS

## Auth
- Cookie session (`web_session` cookie)
- Search and note detail work without login
- User profile may require session or triggers CAPTCHA more aggressively
- Anti-bot: custom obfuscated JS signing (not standard PerimeterX/DataDome/Akamai)

## Transport
- `page` (L3 adapter) — all data extracted from page navigation + SSR state
- Cannot downgrade to `node` — API signing is mandatory and deeply obfuscated

## Extraction
- **Search**: `__INITIAL_STATE__.search.feeds._rawValue` (Vue ref) — array of note cards
- **Note detail**: `__INITIAL_STATE__.note.noteDetailMap[noteId].note` — nested under `.note` when SSR-hydrated
- **Comments**: Intercepted from `waitForResponse` on `/api/sns/web/v2/comment/page`
- **User profile**: `__INITIAL_STATE__.user.userPageData` — contains basicInfo, interactions, tags
- Vue refs: some state values wrapped in `{ _rawValue, _value, dep }` — access via `._rawValue`

## Known Issues
- **CAPTCHA on user profiles**: Rapid page navigation triggers CAPTCHA (`verifyType=124`). Rate-limit based, not auth-based. Resolve by solving CAPTCHA in browser or waiting.
- **Vue reactive wrappers**: State values may be plain objects or Vue refs depending on hydration timing. Adapter handles both via `?._rawValue ?? val` pattern.
- **xsecToken**: Note detail URLs require a `xsec_token` parameter for reliable access. Token comes from search results.
